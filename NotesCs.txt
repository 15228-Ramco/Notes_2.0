 IEnumerable,ICollection,IList
 
 IEnumerable - Basic interface for iteration over collection,allows the use of foreach to iterate over the collection
 ICollection - Implements IEnumerable, Aditonal features to add/remove items to the list of items/objects
 IList 		 - Implements ICollection,Aditonal features to Add/Remove items at a specifc index to the list of items.objects	
 IQueryable  - IQueryable inherits from IEnumerable, so anything you can do with IEnumerable, you can do with IQueryable too. 
				The key difference is where the query is executed:
				With IEnumerable, the query is executed in-memory (on the client). Even if it’s connected to a database (like Entity Framework), 
					it fetches all the data first, and then applies filters in memory.
				With IQueryable, the query is translated into SQL and executed on the server/database, 
					so only the filtered data is retrieved, making it much more efficient for large datasets.
					
					// Assume dbContext.Users is a DbSet<User>

					// IEnumerable: Fetches all users into memory, then filters
					IEnumerable<User> usersEnum = dbContext.Users;
					var filteredEnum = usersEnum.Where(u => u.Name == "John");
					
					// IQueryable: Translates to SQL WHERE clause and filters in DB
					IQueryable<User> usersQuery = dbContext.Users;
					var filteredQuery = usersQuery.Where(u => u.Name == "John");
					filteredEnum causes all users to be loaded into memory, then filtered.

					filteredQuery sends a SQL like SELECT * FROM Users WHERE Name = 'John'.	
					
LINQ queries are often lazily evaluated.
					List<int> numbers = new List<int> { 1, 2, 3, 4, 5 };

					// LINQ query - not executed yet (deferred)
					var evenNumbers = numbers.Where(n => n % 2 == 0);
					
					//Modify the source AFTER query is defined
					numbers.Add(7);
					
					// Now execute the query
					foreach (var num in evenNumbers)
					
					Console.WriteLine(num);
					
		Here 7 is not printed in the foreach because of c# lazy   evalution,
		because the query was not run until the foreach loop.

🚨 Immediate Execution vs Deferred Execution
Type	Executes Immediately?	Example
Deferred	❌ No	.Where(), .Select(), .Skip()
Immediate	✅ Yes	.ToList(), .Count(), .Sum()	

 Why Use Deferred Execution?
✅ Performance: No work is done until needed.

✅ Freshness: Query reflects the most recent data.

✅ Composability: Queries can be built in steps before execution.
